LightWAVE bug list

  • The Tables panel doesn't always show all of the samples in the signal window.
    The implementation of autoplay mode required modifications to the code for collecting samples that have been read from the server. These modifications have been made in the View/edit code, but not yet in the Tables code.
  • Signals not visible/not centered.
    In records with sampling frequencies less than 0.1 Hz, there is at most one sample in the window, and nothing is drawn as a result.
  • Autoplay freezes after >10 minutes.
    It may have been possible to fill the trace pool with redundant copies of the same, or nearly the same, data segment, thereby locking out any attempt to fetch the next data segment. Changes in the prefetch strategy in version 0.46 make this very unlikely. Exit the browser and reopen the record if this problem recurs.
  • LightWAVE doesn't work in Internet Explorer.
    Yes, it does. But unless you have IE 9 or 10, you must install the free Google Chrome Frame plugin.
  • MIMIC II Waveform Database "numerics" records are missing.
    LightWAVE needs a record submenu, like the PhysioBank ATM has.
